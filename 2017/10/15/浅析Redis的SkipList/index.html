浅析Redis的SkipList

发表于 2017-10-15 | | 阅读次数:

SkipList是什么？

SkipList 俗称跳表，是带有层级结构的链表（节点都是排好序的）。通过增加层数，节点上可以带有更多的信息，通过这些信息可以直接访问更远的节点。

一个跳表，应该具有以下特征：

  • 一个跳表应该有几个层（level）组成；
  • 跳表的第一层包含所有的元素；
  • 每一层都是一个有序的链表；
  • 如果元素x出现在第i层，则所有比i小的层都包含x；
  • 第i层的元素通过一个down指针指向下一层拥有相同值的元素；
  • 在每一层中，-1和1两个元素都出现(分别表示INT_MIN和INT_MAX)；
  • Top指针指向最高层的第一个元素。

Skip List作为性能和实现难度的一个折中，它的查询、插入、删除等主要操作时间复杂度期望值也都是 Θ(lgn)，空间复杂度是 Θ(n)。

SkipList查找节点

查找操作很简单，比如我们要找图中节点key为20的节点。

  • 我们首先获取到头节点，从头检点的最高层开始（节点中的点表示指向节点的指针），下一个节点是17, 很明显20>17所以应该在后面，尝试查找level-2层下一个元素。如果继续往后结果是NULL，那说明后面没有要找的节点了。
  • 跳到下一层level-1继续往后是25且20<25说明后面也没有我们要找的节点了。
  • 再跳到下一层level-0，往后就找到我们要的节点了。如果到最下面一层还找不到，那这个节点就肯定不在表中了（因为最低层包含所有节点）。

这里例子中我们只需要2次就找到20这个节点，如果是简单的链表结构则需要5次。

SkipList插入节点

插入操作比较复杂，因为插入操作还要考虑“0-1概率问题”。接下来举例子，假设我们需要插入10节点，需要根据当前表的结构得到一个合适的高度，插入后可以让SkipList的尽量的接近理想的结构，但是实现上这会比较的复杂。

根据概率随机为新节点生成一个高度，具体的规则如下：

  • 给定一个概率p, 产生一个[0,1)之间的随机数
  • 如果这个随机数< p,则向上加一层
  • 重复以上动作，直到随机数>=p

如上图，我们在插入节点之时，从level-0层开始产生随机概率，若满足规则2，则添加level-1层，反之则结束动作。而关于如何选择合适的p，如果一定要给个固定值，可以参考William Pugh的原始论文，但是多数的算法中都是用函数产生一个0~1的随机数p。
